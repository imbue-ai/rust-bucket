# Migration to v0.7.0

## ratchets upgraded to 0.3.1 (ratchets.toml v1 -> v2)

The devcontainer now installs a pinned `ratchets` 0.3.1 (`cargo install ratchets@0.3.1`),
replacing the previous unpinned `install.sh` clone of `main`. ratchets 0.3.1 only accepts
`ratchets.toml` with `[ratchets].version = "2"` and rejects v1 configs at load time, so
every project must migrate its config.

### Steps
1. Rebuild your devcontainer so it installs ratchets 0.3.1.
2. Migrate `ratchets.toml` to the v2 schema:
   - Set `[ratchets].version = "2"`.
   - Rules are now opt-in. Add a top-level `enabled_ratchets` array listing the rule IDs
     and `$set` references (e.g. `$common-starter`) you want enforced.
   - Move any `[rules].rule-id = false` into a top-level `disabled_ratchets` array
     (disabled always wins over enabled).
   - Remove `[rules].rule-id = true`; enablement is expressed only via `enabled_ratchets`.
   - Run `ratchets help upgrade` for the full schema diff.
3. Re-baseline budgets where 0.3.1's detector reports different counts than the previous
   tool: `ratchets bump <rule>` (or `ratchets bump --all`) sets budgets to current counts.
   Tighten them again over time with `ratchets tighten`.
4. Verify the migration: `ratchets check` exits 0.
